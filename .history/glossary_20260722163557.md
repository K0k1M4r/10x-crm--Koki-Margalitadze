# Glossary

## 1. LocalStorage

A browser storage system that allows websites to save data locally on the user's device. In this project, LocalStorage is used to store users, sessions, clients, and theme preferences.

**ქართულად:** ბრაუზერის მეხსიერების სისტემა, რომელიც საშუალებას აძლევს საიტებს შეინახონ მონაცემები მომხმარებლის მოწყობილობაზე. ამ პროექტში გამოიყენება მომხმარებლების, სესიის, კლიენტების და თემის შესანახად.

## 2. API

Application Programming Interface. A way for different software systems to communicate. This project uses an external API to retrieve sample client data.

**ქართულად:** სხვადასხვა პროგრამულ სისტემას შორის კომუნიკაციის საშუალება. პროექტში გამოყენებულია გარე API სატესტო კლიენტების მონაცემების მისაღებად.

## 3. Fetch API

A JavaScript feature used to make HTTP requests. In this project, it is used to request client data and send requests to the external API.

**ქართულად:** JavaScript-ის ფუნქცია, რომლითაც კეთდება HTTP მოთხოვნები. გამოიყენება კლიენტების მონაცემების წამოსაღებად და API-სთან კომუნიკაციისთვის.

## 4. DOM

Document Object Model. A representation of an HTML page that JavaScript can interact with. It is used in this project to update content, create elements, and handle user actions.

**ქართულად:** HTML გვერდის წარმოდგენა, რომელთანაც JavaScript-ს შეუძლია ურთიერთქმედება. გამოიყენება კონტენტის განახლების, ელემენტების შექმნისა და მომხმარებლის ქმედებების დასამუშავებლად.

## 5. Event Listener

A JavaScript function that waits for user actions such as clicks, typing, or form submissions. This project uses event listeners for buttons, forms, search, filters, and theme switching.

**ქართულად:** JavaScript ფუნქცია, რომელიც ელოდება მომხმარებლის ქმედებებს, როგორიცაა დაწკაპუნება ან ფორმის გაგზავნა. გამოყენებულია ღილაკებზე, ფორმებზე, ძებნასა და თემის გადართვაზე.

## 6. Session

A temporary record that stores information about the currently logged-in user. In this project, sessions are stored in LocalStorage to keep users logged in.

**ქართულად:** დროებითი ჩანაწერი, რომელიც ინახავს ინფორმაციას მიმდინარე ავტორიზებული მომხმარებლის შესახებ. ინახება LocalStorage-ში, რომ მომხმარებელი დარჩეს შესული.

## 7. CRUD

Create, Read, Update, Delete. The four basic operations used when managing data. This project uses CRUD operations for client management.

**ქართულად:** მონაცემების მართვის ოთხი ძირითადი მოქმედება - შექმნა, წაკითხვა, განახლება, წაშლა. გამოყენებულია კლიენტების მართვისთვის.

## 8. JSON

JavaScript Object Notation. A format used to store and exchange structured data. This project uses JSON when saving data and communicating with APIs.

**ქართულად:** მონაცემთა სტრუქტურირებული ფორმატი, რომელიც გამოიყენება შესანახად და API-სთან კომუნიკაციისთვის.

## 9. Array Methods

Built-in JavaScript functions for working with arrays, such as map(), filter(), find(), and reduce(). They are used throughout the project for processing client data.

**ქართულად:** JavaScript-ის ჩაშენებული ფუნქციები მასივებთან სამუშაოდ, როგორიცაა map(), filter(), find(), reduce(). გამოყენებულია კლიენტების მონაცემების დასამუშავებლად.

## 10. Authentication

The process of verifying a user's identity. In this project, authentication includes signup, login, session creation, route protection, and logout.

**ქართულად:** მომხმარებლის ვინაობის დამოწმების პროცესი. მოიცავს რეგისტრაციას, ავტორიზაციას, სესიის შექმნას, გვერდების დაცვას და გამოსვლას.
